Step 2: Add the create_box Function

Remember the create_box function from the Collision Detection Challenge? We're going to reuse it in this project! This is a great example of how writing reusable functions makes programming easier.

üîÑ Reusing Code

One of the best practices in programming is to reuse code you've already written. The create_box function creates a table with all the properties we need for collision detection.

Add this function at the TOP of your code (before the other functions):

-- Function to create a new box
function create_box(x, y, size, vx, vy, color)
  local new_box = {
    x = x,
    y = y,
    size = size,
    vx = vx,
    vy = vy,
    color = color,
    original_color = color
  }
  return new_box
end

üîç Understanding the Function

This function creates a table (box) with these properties:

  • x - The x position (center of the box)
  • y - The y position (center of the box)
  • size - The width and height of the box
  • vx - Velocity in the x direction (horizontal speed)
  • vy - Velocity in the y direction (vertical speed)
  • color - The current color
  • original_color - Remembers the starting color (useful for when we change colors during collisions)

üí° Why Add This Now?

We're adding this function first because we'll use it in the next step to refactor how we store the googly eyes. Instead of having separate variables like left_eye_x, left_eye_y, etc., we'll have a single left_eye box that contains all this information!

  3. We haven't changed anything yet, we've just added a helper function

If your game still runs normally, you're ready for the next step!
