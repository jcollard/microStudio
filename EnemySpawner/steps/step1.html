Before building the dynamic spawning system, you need a project with the basic enemy system in place. This lesson builds on the foundation from the Abstracting Enemies lesson.

Option 2: Continue from Your Own Project

If you completed the Abstracting Enemies lesson, you can continue with your existing project. Just make sure it meets the requirements below.

Requirements Checklist

Your project must have the following before proceeding:

  • An enemies table - A table that stores all enemy objects
  • create_enemy() factory function - Creates and returns a new enemy object with properties like:
    • x, y - Position
    • isDestroyed - Boolean flag for destroyed state
    • update() method - Updates enemy behavior
    • draw() method - Renders the enemy
  • init_enemies() function - Initializes enemies at game start
  • update_enemies() function - Updates all active enemies each frame
  • draw_enemies() function - Draws all active enemies each frame

Why These Requirements?

This lesson will modify these existing systems to add dynamic spawning. You'll enhance init_enemies() to use a spawning function, create new spawning logic, and add wave mechanics on top of what you already have.

Prerequisites

You should be comfortable with:

  • Tables and iteration - Using for loops with pairs() and traditional counting loops
  • Object properties - Accessing and modifying enemy properties like enemy.x, enemy.isDestroyed
  • Factory functions - Calling create_enemy() to generate new objects
  • Boolean logic - Checking conditions with if statements
  • Game loop concepts - Understanding init(), update(), draw()

What Your Project Should Look Like

Before starting this lesson, your project should function like this demo. You should be able to shoot and destroy enemies, but once they're all gone, the game is over.

Use arrow keys to move, space to shoot. Notice how the game ends once all enemies are destroyed.
