Step 2: Add `isDestroyed` Property to Eyes

To implement object destruction, we need a way to track whether an eye has been destroyed. We'll add an isDestroyed property to each eye.

üîç Understanding Object State

An object's state is a collection of properties that describe how it is right now. We can use state to control behavior:

  • If isDestroyed = false ‚Üí the eye is alive and moves normally
  • If isDestroyed = true ‚Üí the eye is destroyed (we'll skip updating and drawing it)

üí° Why Not Just Delete the Eye?

You might think we should just remove the eye from the game when it's destroyed. But that's dangerous!

If we delete during a loop (like collision checks), the loop breaks. Instead, we "mark as destroyed" and skip it in loops. This is called lazy removal.

Your Task: Add isDestroyed Properties

Find your init_eyes() function and add one line to each eye:

function init_eyes()
  eye_rotation = 0

  left_eye = create_box(0, 0, 28, 28, 0.7, 0.57575, "#FF00FF")
  left_eye.isDestroyed = false  -- NEW: Track if destroyed

  right_eye = create_box(0, 0, 28, 28, -0.6, 0.37575, "#00FFFF")
  right_eye.isDestroyed = false  -- NEW: Track if destroyed
end

üìù What You're Doing

  • Adding a property to track state
  • Initializing it to false (eyes start alive)
  • This property will be checked in update and draw functions
