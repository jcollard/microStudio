Step 3: Skip Movement for Destroyed Eyes (Part 1 - Left Eye)

Now we'll use the isDestroyed property to control behavior. When an eye is destroyed, it should stop moving and stop bouncing.

üí° The Power of State

By checking if not left_eye.isDestroyed then before updating movement, we can control the entire behavior with one property!

Your Task: Wrap Left Eye Movement

Find your update_eyes() function and wrap all the left eye movement code:

function update_eyes()
  eye_rotation = eye_rotation + 12

  -- Left eye - ONLY UPDATE IF NOT DESTROYED
  if not left_eye.isDestroyed then
    -- Move the eye
    left_eye.x = left_eye.x + left_eye.vx
    left_eye.y = left_eye.y + left_eye.vy

    -- Boundary checks
    if left_eye.x < -100 then left_eye.vx = 0.7 end
    if left_eye.x > 100 then left_eye.vx = -0.7 end
    if left_eye.y < -100 then left_eye.vy = 0.57575 end
    if left_eye.y > 100 then left_eye.vy = -0.57575 end
  end

  -- Right eye (unchanged for now)
  right_eye.x = right_eye.x + right_eye.vx
  right_eye.y = right_eye.y + right_eye.vy
  -- ... rest of right eye code ...
end

üìù What You're Doing

  • Adding an if not left_eye.isDestroyed then check
  • Moving all left eye movement code inside this check
  • The right eye stays unchanged (we'll update it in the next step)
  • Indentation is important for readability!

üß™ Test Your Code

After making the change, test it in the console:

left_eye.isDestroyed = true

Watch what happens:

  • The left eye should stop moving immediately
  • It won't bounce off the boundaries anymore
  • The right eye keeps moving normally

Now try:

left_eye.isDestroyed = false

The left eye should start bouncing again!
